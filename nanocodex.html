01设计思想

nanocodex 不是要做「功能最全的 Agent」，而是用最少的概念把一个编码 Agent 的关键链路跑通、且每一步都可解释、可测试、可信任。整套代码贯穿四条原则。

核心循环刻意精简

一个回合就是「调用模型 → 执行工具 → 把结果喂回 → 继续」直到任务完成。Agent 不知道控制台、不知道 GUI，渲染靠 hook 注入，循环本身只关心推进任务。

纯函数 + 可注入 I/O

schedule、memory、skills、compaction、pricing、marketplace 的核心逻辑都是对数据的纯函数，时间、网络、文件都从外部注入。结果是整套 420 个测试全离线、不碰真实 API 与网络。

对局限性诚实

价格表是硬编码快照、带 as-of 日期，未知模型返回「成本未知」而不是错数；Windows 沙箱明说是策略级、非内核隔离；视觉模型支持标注为未验证。不假装能力。

无隐藏状态

会话是可读的 JSONL、记忆是用户自己的 markdown 文件、定时任务是一份 JSON。没有黑盒数据库，用户能直接看到、编辑、删除 Agent 依赖的一切。

02核心：一个回合的循环

AgentLoop.run_turn() 驱动单个用户回合直到完成。一个回合内可能发生多轮模型调用（每次工具使用算一轮），token 用量逐轮累计、折算成本。

用户输入 → 调用模型 → 解析工具调用 → 沙箱/审批闸门 → 执行工具 ↺ 回合完成 
# agent/loop.py — 回合产物
class TurnResult:
    final_text: str
    iterations: int          # 本回合循环了几轮
    stop_reason: str
    tools_used: list[str]
    usage: dict[str, int]    # 真实 token 累计 → 折算成本

03核心能力

能力做法
模型后端DeepSeek / 任意 OpenAI 兼容 /v1/chat/completions，支持流式；改 base_url 即可指向本地模型（vLLM / llama-server / LM Studio）。
工具集shell · apply_patch · update_plan · read_file · web_search · manage_schedule · manage_skills · remember，统一经 ToolRegistry 派发。
沙箱策略三种模式 read-only / workspace-write / danger-full-access，决定可写根目录与网络；策略与执行分离。
审批状态机四档 untrusted / on-failure / on-request / never，越权操作（含联网搜索）走 ASK / AUTO_DENY / AUTO_APPROVE。
上下文压缩deterministic（零成本规则摘要，默认）或 summarizer（消耗 token，opt-in）；保留 system + 近期尾部，且尾部必须从 user 消息开始。
自适应推理档按用户输入选 max / low / high；关键词表覆盖中日英（"报错""调试""デバッグ" 都能命中）。
成本统计cache-aware：命中 token 比未命中便宜约 120×，有拆分时分别计费，无拆分时按未命中保守计，从不低估。
会话历史JSONL 持久化；按 session_id 存全量快照，可在 GUI 左栏点选回放真实对话；支持 --resume。
记忆 / AGENTS.md / Skillsmemory 记「谁/什么」（用户偏好），skills 记「怎么做」，AGENTS.md 是项目级指令；skills 只注入名称+描述，正文按需读取。
MCP + 市场来自 ~/.nanocodex/mcp.toml，以 mcp__<server>__<tool> 暴露；运行在沙箱外，内置 catalog + 远程 catalog（按不可信数据处理）。
GUI 特性prompt 增强（✨）、# 快速记忆、定时任务、A/B worktree 对比（两套配置同 prompt 各跑一遍，对比 diff/成本/延迟，采纳其一）。

05快速开始

# 安装（含开发依赖）
python -m pip install -e ".[dev]"

# CLI
nanocodex --cd .
nanocodex "add a --json flag to the CLI"

# GUI
nanocodex-gui --cd .

# 全套离线测试（无需真实 key / 网络）
python -m pytest -q
安全：真实 API key 永远留在仓库外（环境变量或 ~/.nanocodex/config.toml）。Windows 沙箱是策略级而非内核隔离；MCP 工具在沙箱外执行子进程，只启用你信任的服务。
